# ORIGO

المتجر يعمل مباشرة من `index.html`. ولتفعيل مصدر **OpenAI + بحث الويب** شغّله من خلال الخادم المحلي الآمن حتى لا يظهر مفتاح API داخل المتصفح:

```powershell
$env:OPENAI_API_KEY="ضع_المفتاح_هنا"
.\start-origo.ps1
```

ثم افتح:

```text
http://127.0.0.1:4173
```

يمكن اختيار موديل آخر اختياريًا:

```powershell
.\start-origo.ps1 -Model "gpt-5.4-mini" -Port 4173
```

أو استخدم `npm start` عند توفر Node.js وnpm. إذا كان المنفذ `4173` مستخدمًا، اختر منفذًا آخر في الأمر ثم افتح المنفذ نفسه في المتصفح.

مفتاح OpenAI API منفصل عن اشتراك ChatGPT ولا يُحفظ في `localStorage` أو ملفات الواجهة. بدون المفتاح تستمر المصادر العامة الحالية والمسودة اليدوية في العمل.

## قاعدة معرفة المكونات

بيانات المكونات التي قدّمها مدير المتجر تُبنى في ملف محلي منظّم:

```powershell
npm run build:knowledge -- "C:\path\to\pasted-text.txt"
```

الملف الناتج `fragrance-knowledge.js` يُحمّل قبل نظام مصادر الكتالوج ويُستخدم لتصنيف النوتات وصياغة وصف أولي عند وجود مكونات مؤكدة.
