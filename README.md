# ORIGO

متجر عطور ثنائي اللغة مع حسابات وسلة وطلبات ولوحة إدارة ومكتبة مكونات عطرية مترابطة مع المنتجات.

## التشغيل والتحقق

يتطلب Node.js 20 أو أحدث:

```powershell
pnpm install
pnpm test
pnpm start
```

يمكن أيضاً استخدام `npm install` و`npm start`. لا يوجد وضع `watch` ضمن أوامر المشروع.

## النشر على Hostinger Node.js

المشروع لا يستخدم `node:sqlite` ولا أي إضافة قاعدة بيانات Native. طبقة البيانات تعتمد على `sql.js` (SQLite WebAssembly)، لذلك تعمل على استضافة Node.js المشتركة من دون الحاجة إلى إعادة بناء مكتبات C/C++.

إعدادات Hostinger المقترحة:

- Startup file: `server.mjs`
- Node.js: الإصدار 20 أو أحدث
- Install command: `npm install --omit=dev`
- Start command: `npm start`
- اجعل مجلد البيانات دائماً، واضبط `ORIGO_DB_PATH` على مسار كامل قابل للكتابة مثل `/home/USER/origo-data/origo.db`.
- يستخدم الخادم `PORT` الذي توفره الاستضافة تلقائياً.

متغيرات البيئة:

```text
ORIGO_DB_PATH=/home/USER/origo-data/origo.db
ORIGO_ADMIN_EMAIL=admin@example.com
ORIGO_ADMIN_PASSWORD=use-a-long-unique-password
ORIGO_ADMIN_NAME=ORIGO Admin
OPENAI_API_KEY=optional
OPENAI_MODEL=gpt-5.4-mini
```

لا تضع كلمات المرور أو مفاتيح API داخل المستودع. قاعدة البيانات مناسبة لتشغيل نسخة واحدة من تطبيق Node.js، وهو النمط المعتاد للاستضافة المشتركة.

## مكتبة المكونات العطرية

- صفحة المكتبة: `/notes`
- صفحة المكون: `/notes/rose` أو أي Slug آخر
- بحث عربي/إنجليزي وفلترة حسب العائلة
- مرادفات موحّدة مثل `Oud` و`Oudh` و`Agarwood` و`عود`
- صور وبيانات المكونات تُجلب تلقائياً من المكتبة عند كتابة أسماء النوتات في المنتج
- المكونات غير المعروفة تُضاف إلى قائمة المراجعة بدلاً من تعطيل المنتج
- صفحات المكونات تضبط العنوان والوصف وCanonical وبيانات JSON-LD تلقائياً

تُدار المكتبة من لوحة الإدارة، وتشمل إضافة العائلات والمكونات، الصور، المرادفات، الموضع العطري، التصنيف والدمج.

## الكتالوج ولوحة الإدارة

تُنشأ حساب الإدارة عند أول تشغيل من متغيرات `ORIGO_ADMIN_*`. لوحة الإدارة تشمل المنتجات والطلبات والعملاء والمخزون ومكتبة المكونات وبقية أقسام التشغيل. البيانات التي تحتاج مزوداً خارجياً تبقى API-ready إلى أن تُضاف بيانات اعتماد ذلك المزود.

الوظائف التشغيلية المتاحة تشمل:

- نسخ المنتج، نشره، إيقافه وأرشفته.
- تحديث حالة الطلب والدفع وشركة الشحن ورقم التتبع والملاحظات الداخلية.
- سجل زمني للطلب وطباعة الفاتورة وبوليصة الشحن.
- إنشاء حسابات الموظفين ضمن 11 دوراً إدارياً، مع فرض الصلاحيات على مسارات API وسجل نشاط.
- حفظ بيانات أقسام لوحة الإدارة في قاعدة الخادم مع نسخة محلية احتياطية للواجهة.
- تصدير التقارير بصيغ CSV وExcel والطباعة إلى PDF.

تكاملات شركات الدفع والشحن والإعلانات وWhatsApp تحتاج مفاتيح وحسابات من المزود المختار؛ الواجهات والحقول جاهزة، لكن إرسال العمليات إلى مزود خارجي لا يبدأ قبل إضافة بيانات اعتماده.

عند إضافة عطر يمكن إدخال أسماء المكونات فقط داخل:

```text
Top Notes: Rose, Bergamot
Heart Notes: Jasmine, Iris
Base Notes: Oud, Musk, Vanilla
```

ويحتفظ الخادم بمواضع الافتتاحية والقلب والقاعدة، بينما توفر المكتبة الاسم العربي والعائلة واللون والوصف والصورة.

## قاعدة معرفة المكونات

لإعادة بناء ملف المعرفة من ملف نصي يقدمه مدير المتجر:

```powershell
npm run build:knowledge -- "C:\path\to\pasted-text.txt"
```

ينتج الأمر `fragrance-knowledge.js` ويستخدم لتصنيف النوتات وصياغة مسودات وصف أصلية.
